# SourceTree

## Git的概念

* Git 是一个分布式版本控制系统，用于追踪文件变更，支持多人协作。每个开发者的本地都拥有完整的仓库历史，不依赖中央服务器即可进行提交、分支、合并等操作。

## 三个区域的概念

* 工作区
     * 电脑上能直接看到、直接操作项目文件夹，在此编辑文件
* 暂存区
     * 一个临时存放修改的区域，通过 `git add` 将工作区的改动“暂存”起来，准备提交
* 版本库
     * Git 用来永久存储项目历史版本的地方。它包含了所有提交过的版本数据，分支信息，以及 Git 管理项目所需的一切数据。

## 仓库

*  Git 管理的项目根目录，内含 `.git` 隐藏文件夹，记录了所有版本信息和配置。仓库可以是本地仓库，也可以是远程仓库（存在于服务器）。仓库与仓库之间通过 远程关联进行同步。

## 节点

* `git`使用 `commit`之后就会生成一个节点，节点包含：作者、时间、日志信息、变更内容、父节点指针（形成链状历史）。节点是版本回溯、分支合并、差异比较的基本单元。

## 分支

* 指向某个节点的可变指针，用于隔离开发线。默认主分支通常为 `master` 或 `main`。分支允许并行开发新功能、修复 bug，互不干扰。
